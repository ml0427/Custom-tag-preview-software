以下是為開發專屬 ZIP 漫畫管理軟體規劃的階段性任務表。

### 階段一：專案初始化與基礎建設
* [ ] **後端環境建置**：初始化 Spring Boot 專案，加入 Web、JPA/MyBatis 依賴。
* [ ] **資料庫設定**：配置 SQLite 連線，確保單機運行環境。
* [ ] **前端環境建置**：初始化 Vue 專案，設定路由 (Vue Router) 與狀態管理 (Pinia/Vuex)。
* [ ] **目錄結構規劃**：在本地端定義好「ZIP 漫畫存放區」與「封面快取區」的實體資料夾路徑。

### 階段二：資料庫設計與核心檔案處理
* [ ] **資料表建立**：
    * 建立 `comic` 表 (欄位：id, 檔案路徑, 書名, 自訂封面內部路徑, 匯入時間)。
    * 建立 `tag` 表 (欄位：id, 標籤名稱)。
    * 建立 `comic_tag_mapping` 表 (欄位：comic_id, tag_id)。
* [ ] **ZIP 讀取邏輯**：使用 Java `ZipFile` 撰寫工具類別，實作取得 ZIP 內檔案列表及過濾圖片檔的功能。
* [ ] **圖片提取邏輯**：實作從 ZIP 中提取第一張圖片或指定檔名圖片的位元組陣列 (Byte Array)。

### 階段三：非同步處理與效能優化
* [ ] **背景掃描機制**：開發掃描本地目錄的邏輯，將新發現的 ZIP 檔資訊寫入資料庫。
* [ ] **非同步任務服務**：實作非同步處理機制，在背景批次執行 ZIP 封面提取與縮圖產生，避免阻塞主程式。
* [ ] **快取寫入**：將提取出的封面圖片轉存至本地「封面快取區」，並命名為對應的漫畫 ID。

### 階段四：後端 API 開發
* [ ] **漫畫列表 API**：實作查詢與分頁功能，支援透過標籤 ID 進行篩選 (`JOIN` 查詢)。
* [ ] **標籤管理 API**：實作標籤的新增、刪除，以及為特定漫畫綁定/解除標籤的功能。
* [ ] **封面讀取 API**：實作靜態資源伺服邏輯，優先回傳快取資料夾的圖片；若無快取，則觸發單次 ZIP 讀取並回傳串流。
* [ ] **自訂封面 API**：接收前端指定的 ZIP 內圖片路徑，更新 `comic` 表的自訂封面欄位，並觸發非同步任務更新快取。

### 階段五：前端介面與互動實作
* [ ] **主畫面圖庫**：開發網格狀的畫廊介面 (Grid Layout)，展示漫畫封面縮圖與書名。
* [ ] **標籤篩選器**：在畫面上方或側邊實作標籤列表，點擊後觸發 API 重新載入漫畫清單。
* [ ] **漫畫詳情與標籤編輯**：開發彈出視窗或獨立頁面，顯示單本漫畫資訊，並提供新增/移除該漫畫標籤的 UI。
* [ ] **內容預覽與設定封面**：
    * 實作請求並展開 ZIP 內所有圖片列表的介面。
    * 在每張圖片旁實作「設為封面」按鈕，點擊後呼叫自訂封面 API 並重新載入主畫面縮圖。

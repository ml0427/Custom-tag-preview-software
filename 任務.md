# 專案交接與開發筆記 (Handover Notes)

這份文件用於記錄目前的系統架構、已完成進度與未來接手的開發重點，方便後續 AI 或是開發人員能夠無縫接軌。

## 1. 系統環境與技術棧
* **前端 (Client)**：
  - 目錄：`client/`
  - 框架：Vue 3 + Vite + TypeScript (由 Webview2 渲染)
* **後端 (Server)**：
  - 目錄：`server/`
  - 框架：Spring Boot 3.4.2 (Java 17)
  - 資料庫：SQLite (`server/comic.db`)
  - 啟動方式：`.\mvnw spring-boot:run`
* **實體存放區**：
  - `comic_storage/`：存放來源 ZIP 漫畫檔。
  - `comic_cache/`：存放封面快取圖片。

## 2. 已完成進度 (STAGE 1 ~ 7)
1. **環境建置**：完成基礎專案結構與 SQLite 連線設定。
2. **資料庫設計**：建立 `Comic`, `Tag` 多對多關聯模型。
3. **核心檔案處理**：實作 `ZipService` 提取圖片。
4. **掃描與快取**：實作手動觸發式掃描與非同步封面快取。
5. **後端 API 服務**：提供分頁、標籤過濾、重新命名與封面重取接口。
6. **一鍵啟動**：建立 `start.bat` 與 `啟動說明.md`。
7. **【今日重點】檔案總管介面與自動化分類**：
   - **UI 改版**：將原本的 Grid 改為「資料清單 + 右側預覽窗格」，並支援**可拖動分隔線 (Resizer)**。
   - **鍵盤導覽**：支援 `上下鍵` 選取、`Enter` 開啟詳情、`F2` 快速重新命名。
   - **檔案管理**：實作清單**行內編輯 (Inline Rename)**，支援檔名左側編輯按鈕。
   - **自動標籤 (Auto-Tagging)**：利用正則表達式自動從 `[...]` 或 `【...】` 提取作者，且支援 `xxx(oooo)` 複雜格式拆分。
   - **掃描同步補強**：實作了「掃描前先行清空資料庫與快取」的機制，解決舊有紀錄標籤不完全的問題。

## 5. PM / UX 審查紀錄（2026-04-24）

### Product Manager 觀察

**核心價值沒有被強調**
- 標籤系統是最大差異點，卻與工作目錄並列，感覺像附加功能
- `api.mergeTags()` 已實作，但 TagSidebar 無入口
- 新增標籤只能靠系統選單 `menu-new-tag`，沒有明顯的「＋」按鈕
- 掃描標籤 Wizard 藏在工作目錄面板，切換到標籤模式就消失

**缺少進度感知**
- 標籤清單無筆數，無法判斷哪些未標記、哪個標籤最常用

### UI/UX 觀察

| # | 問題 | 說明 |
|---|------|------|
| 1 | 空狀態引導不足 | 首次進入只顯示文字提示，無 onboarding 步驟 |
| 2 | 標籤無數量顯示 | 應加筆數如 `# 作者名 (12)` |
| 3 | 原生 `alert/confirm` 破壞視覺 | 與深色主題不搭，是整體質感最大破口 |
| 4 | 檔案列表無標籤狀態 | 無法一眼看出哪些已標記 |
| 5 | 預覽窗格切換按鈕太小 | 20px 寬的 `‹/›`，新用戶不易發現 |
| 6 | ActivityBar 只有兩項但佔 64px | 略顯空曠 |

### 建議優先處理

| 優先 | 項目 | 理由 |
|------|------|------|
| 高 | `alert/confirm` 換成 in-app toast/dialog | 每次操作都會碰到，影響整體質感 |
| 高 | 標籤清單加筆數 | 低成本、高資訊密度 |
| 中 | 檔案列表加標籤 chip 顯示 | 讓核心功能更可見 |
